# SOLID Principles

Core object-oriented design heuristics for maintainable code.

- **(S) Single Responsibility**: A class should have one, and only one, reason to change.
- **(O) Open–Closed**: Open for extension, closed for modification.
- **(L) Liskov Substitution**: Subtypes must be substitutable for their base types without altering program correctness.
- **(I) Interface Segregation**: Many client-specific interfaces are better than one general-purpose interface.
- **(D) Dependency Inversion**: Depend on abstractions, not concretions.
